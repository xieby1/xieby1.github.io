Introduction

MAPLE IR is an internal program representation to support program compilation and execution.Â  Because any information in the source program may be useful for program analysis and optimization, MAPLE IR aims to provide information about the source program that is as complete as possible.

Program information consists of two parts: declaration part for definingÂ the program constructs and theÂ executable program code.Â  The former is commonly referred to as theÂ symbol table.

MAPLE IR is target-independent.Â  It is not pre-disposed towards any specific target processor or processor characteristic.

MAPLE IR can be regarded as the ISA of a virtual machine (VM).Â  The MAPLE VM can be regarded as a general purpose processor thatÂ takes MAPLE IR as input and directly executes the program portion of the MAPLE IR.Â  MAPLE VM can be regarded as the first consumer of MAPLE IR.Â  A program compiled into MAPLE IR can be executed on MAPLE VM without the need to finish its compilation to the machine instructions of any target processor.

MAPLE IR is the common representation for programs compiled from different programming languages, which include general purpose languages like C, C++ and Java.Â Â MAPLE IR is extensible.Â  As additional languages, including domain-specific languages,Â are compiled into MAPLE IR, more constructs will be added to represent constructs unique to each language.

MAPLE IR is capable of supporting all known program analyses and optimizations, owing to its flexibility of being able to represent program code at different semantic levels. Â  MAPLE IR's program representation at the highest level exhibits the following characteristics:

  • Many language constructs

  • Short code sequences

  • Constructs are hierarchical

  • No loss of program information

Language-specific analyses and optimizations are best performed at the high level.Â  As compilation proceeds, MAPLE IR is gradually lowered so that the granularity of its operations corresponds more closelyÂ to the instructions of a general purpose processor.Â  At the lower levels, general purpose optimizations are performed.Â  In particular, at the lowest level, MAPLE IR instructions map one-to-one to machine instructions most of the time, for the mainstream processor ISAs.Â  This is important to maximize the effects of optimizations at the IR level, as each eliminated operation will have the corresponding effect on the target machine.Â  At the lowest level, all operations, including type conversion operations, are explicitly expressed at the IR level.Â 

MAPLE IR represents program code intrinsicallyÂ in the form of trees.Â  At the highest level, it honors the hierarchical form of the program as it exists at the source level via the tree representation.Â  It also honors the abstract operations defined by the language.Â  As compilation proceeds, the abstract operations are lowered into general-purpose operations that require longer code sequences.Â  The program structure also becomes more flat, as general-purpose processors work by executing lists ofÂ instructions sequentially.

Since MAPLE IR is one IR that can exist at multiple levels of semantics, the level of a MAPLE IR program is dictated by the constraints that it adheres to.Â  These constraints are of the following two types:

  • Opcodes allowed - The higher the level, the more types of opcodes allowed, including opcodes generated only from specific languages.Â  At the lowest level, only opcodes that correspond one-to-one to operations in a general purpose processor are allowed.

  • Code structure - The program structure is hierarchical at the higher levels.Â  The hierarchical constructs become less and less as lowering proceeds.Â  At the lowest level, the program structure is flat, consisting of sequences of primitive instructions consumed by the general purpose processor.

Though MAPLE IR is target-independent at the highest level, the lowering process will make it become target-dependent.

The type field in the expression node gives the type associated with the result of the operation.
